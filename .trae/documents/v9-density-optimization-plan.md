# v9 图片信息密度优化计划

> **目标**：解决当前v9生成图片信息量不足、文字过度压缩、页面空白过多等问题，使图片信息密度达到优质小红书笔记水平。

---

## 一、问题诊断：当前图片 vs 优质笔记对比

### 1.1 信息量对比（核心差距）

| 维度 | 当前v9输出 | 藤簪语优质笔记 | 差距 |
|------|-----------|--------------|------|
| **总页数** | 7张 | 15-18张 | ⚠️ 页数偏少 |
| **阶段数量** | 3个 | 4个 | 🔴 少1个阶段 |
| **子话题总数** | 5个 | 8-10个 | 🔴 少3-5个话题 |
| **每页论述数** | 5-6条 | 8-12条 | 🔴 密度不足 |
| **论述字数/条** | 30-40字 | 40-60字 | ⚠️ 偏短 |
| **页面空间利用率** | ~45% | ~80% | 🔴 大量空白 |

### 1.2 具体问题清单

#### 问题1：概要页（summary_outline.html）空间浪费严重
- **现象**：页面下半部分约55%是空白
- **原因**：只有3个阶段5个话题，内容太少撑不满页面
- **影响**：用户滑到第2页就看到大量空白，体验差

#### 问题2：内容页（content_detailed.html）信息密度低
- **现象**：
  - 每页只有2个子话题，每个话题3条论述
  - 页面下半部分约50%是空白
  - Stage 3 只有1个话题3条论述，几乎半页空白
- **原因**：
  - 字体偏大（topic-title 17px, point-text 15px）
  - padding/margin 过大（topic-block padding 12px 14px, gap 12px）
  - LLM生成的结构化内容本身不够丰富（仅3阶段5话题）

#### 问题3：LLM生成的内容结构不够细分
- **现象**：
  - 3个阶段中，Stage 3 只有1个话题
  - 每个话题只有3条论述
  - 总共只有14条论述点
- **原因**：Prompt 中对"每个阶段下2-4个子话题"的约束不够强，LLM倾向于少生成

#### 问题4：字体和间距设置偏向"宽松"而非"密集"
- **现状**：
  - body padding: 36px 42px → 偏大
  - topic-block padding: 12px 14px → 偏大
  - topic-title font-size: 17px → 可缩小
  - point-text font-size: 15px → 可缩小至14px
  - points-list gap: 6px → 可缩小至4px
  - body gap: 12px → 可缩小至8px

## 二、优化方案

### 优化方向1：Prompt 层面 — 提升内容丰富度

#### 修改 v9 Prompt 的内容量要求

**当前要求**：
- 3-5 个阶段
- 每个阶段 2-4 个子话题
- 每个子话题 3-5 条论述

**优化后要求**：
- **4-5 个阶段**（强制至少4个）
- 每个阶段 **3-4 个子话题**（强制至少3个）
- 每个子话题 **4-5 条论述**（强制至少4条）
- 论述字数 **40-60字**（从30-50提升）
- 新增约束：**禁止任何阶段少于2个话题**
- 新增约束：**总论述数不少于20条**

### 优化方向2：HTML模板层面 — 提升空间利用率

#### summary_outline.html 优化

| 参数 | 当前值 | 优化值 | 说明 |
|------|-------|--------|------|
| body padding | 40px 45px | 32px 38px | 减少边距 |
| header margin-bottom | 28px | 20px | 减少标题区高度 |
| header-title font-size | 30px | 26px | 缩小标题 |
| stage-item padding | 14px 18px | 11px 14px | 缩小卡片内边距 |
| stage-item gap | 14px | 10px | 缩小卡片间距 |
| stage-title font-size | 18px | 16px | 缩小标题 |
| topic-item font-size | 15px | 14px | 缩小话题字号 |
| topics-list gap | 5px | 4px | 缩小话题间距 |

**新增**：在概要页底部添加"核心数据摘要"区域，展示3-4个关键数据点，填充底部空白。

#### content_detailed.html 优化

| 参数 | 当前值 | 优化值 | 说明 |
|------|-------|--------|------|
| body padding | 36px 42px | 30px 36px | 减少边距 |
| header margin-bottom | 16px | 12px | 减少页眉高度 |
| stage-title font-size | 22px | 20px | 缩小阶段标题 |
| time-range margin-bottom | 14px | 10px | 减少时间标签间距 |
| body gap | 12px | 8px | 缩小子话题间距 |
| topic-block padding | 12px 14px | 10px 12px | 缩小卡片内边距 |
| topic-title font-size | 17px | 15px | 缩小话题标题 |
| topic-title margin-bottom | 8px | 6px | 减少标题间距 |
| points-list gap | 6px | 4px | 缩小论述间距 |
| tag font-size | 13px | 12px | 缩小标签字号 |
| point-text font-size | 15px | 14px | 缩小正文字号 |

**新增**：支持单个页面容纳更多子话题（3-4个而非2个），通过缩小字体和间距实现。

### 优化方向3：ImageGenerator 层面 — 智能分页策略

#### 当前分页逻辑
- 概要页：1页（固定）
- 内容页：每阶段1页（固定）
- 思考页：1页（固定）
- 出处页：1页（固定）

#### 优化后分页逻辑
- **概要页**：1页（固定，但增加数据摘要）
- **内容页**：按**子话题数量**动态分页
  - 如果一个阶段有3-4个话题，拆分为2页（每页2个话题）
  - 单页最多容纳 **3个子话题 × 4-5条论述 = 12-15条论述**
  - 当话题内论述超过5条时，自动换行或拆分
- **思考页**：1页（固定）
- **出处页**：1页（固定）

### 优化方向4：视觉层次增强

#### 新增元素

1. **关键数据卡片**（概要页底部）
   - 展示3-4个最重要的数据点
   - 使用数字+说明的紧凑格式
   - 示例："📊 年化收益 8%-10%" "📉 最大回撤 <10%"

2. **论述密度指示器**
   - 每页显示本页论述数量
   - 格式："本页 12 条论述"

3. **标签颜色区分**
   - 【痛点】→ 红色系 (#e53e3e)
   - 【决策】→ 蓝色系 (#3182ce)
   - 【结果】→ 绿色系 (#38a169)
   - 【数据】→ 橙色系 (#dd6b20)
   - 【案例】→ 紫色系 (#805ad5)
   - 【行动】→ 青色系 (#319795)
   - 【人物】→ 粉色系 (#d53f8c)
