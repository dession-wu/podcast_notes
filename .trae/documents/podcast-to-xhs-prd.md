## 一、项目概述

### 1.1 产品定位

**产品名称**：播客笔记（Podcast Notes）

**产品定位**：一款面向知识工作者与内容创作者的播客深度提炼与内容重塑（Repurposing）引擎。基于大语言模型，将非结构化的长音频转化为高价值的结构化知识库，并支持一键渲染为多场景社交图文资产（支持小红书、朋友圈、公众号等格式）。

**一句话描述**：不是帮你"抄"播客，而是帮你"读懂"播客——让每一期播客都成为你个人知识资产的一部分。

### 1.2 核心价值主张

我们不做"全自动流水线"，而是做你的**创作副驾（Copilot）**——深度理解内容，而非简单摘要；沉淀知识资产，而非阅后即焚。

| 维度 | 市面上的工具 | 我们的产品 |
|-----|------------|-----------|
| **内容理解** | 干瘪的 Summary，罗列要点 | 带有因果逻辑、情绪价值、金句提炼的多维解析（v1-v9 模板体系） |
| **创作模式** | 一键生成，不可干预 | 人机协同：AI 提供结构化素材，用户终审与风格化调整 |
| **价值沉淀** | 阅后即焚，用完即走 | 固化为可检索、可复用的个人图文数字资产 |
| **使用门槛** | 收费 SaaS、强制注册 | 本地部署、完全免费、无需注册、隐私安全 |

### 1.3 项目背景

当前播客内容消费市场中存在两个显著痛点：

1. **播客听众侧**：大量优质的播客内容（商业分析、投资理财、知识分享类）缺乏高效的回顾方式。听众听完后想回顾核心观点时，只能靠记忆或反复回放，效率极低。

2. **内容创作者侧**：播客是极佳的内容素材源，但将播客内容转化为小红书等平台的图文笔记需要大量的手动整理工作——听音频、记笔记、提炼观点、排版配图，整个过程可能需要数小时。

本项目通过结合 Whisper 语音识别技术和大语言模型的内容理解能力，提供了一套完整的自动化解决方案。

### 1.4 项目目标

**MVP 阶段目标**：
- 跑通"音频上传 → 语音转写 → 内容提炼 → 图文笔记生成"核心链路
- 支持 3 种笔记模板风格（标准型/深度干货型/故事共鸣型）+ 图文结构化输出
- 支持本地部署，保证数据隐私

**长期目标**：
- 接入 MCP 架构与 RPA 实现自动发布
- 建立全自动定时内容矩阵
- 扩展更多平台和输出格式

## 二、用户需求分析

### 2.1 目标用户画像

#### 用户画像 A：知识型内容创作者（核心用户）

**基本特征**：
- 年龄：25-35 岁
- 职业：自媒体博主、知识类内容创作者、垂直领域博主
- 使用平台：小红书、公众号、B站、即刻、朋友圈
- 技术水平：具备基本的电脑操作能力，愿意尝试新工具

**核心需求**：
- 需要大量优质的内容素材来维持更新频率
- 播客是重要的素材来源，但整理效率太低
- 希望内容既有深度又能吸引受众，需要多种风格适配不同选题
- 注重内容合规性，需要注明来源避免版权风险

**使用场景**：
> 小林是一个投资理财类的内容创作者，每周需要在多个平台发布 3-5 篇笔记。她发现很多优秀的商业播客（如"面基"、"纵横四海"）的内容质量很高，但每次听完一集60分钟的播客后，手动整理出可用的内容需要花2-3小时。使用播客笔记后，她上传音频文件，5分钟后就能获得3-4种不同风格的高密度社交图文草稿，只需要简单修改就能直接发布。

#### 用户画像 B：播客重度听众（次级用户）

**基本特征**：
- 年龄：22-40 岁
- 职业：学生、职场人士、自由职业者
- 使用习惯：每天收听播客1-3小时，涵盖商业、科技、个人成长等领域

**核心需求**：
- 听完播客后想要回顾核心观点，但不想反复回放
- 希望有一个结构化的笔记帮助理解和记忆
- 偶尔需要引用播客内容写文章或做分享

**使用场景**：
> 小张是一位产品经理，每天通勤路上会听1小时播客。听完一期关于产品设计的深度讨论后，他想要快速回顾嘉宾提到的几个关键方法论。上传音频到播客笔记后，他得到了一个结构清晰的概要，包含了每个核心观点和对应的金句引用。

#### 用户画像 C：研究人员与学习者

**基本特征**：
- 年龄：20-50 岁
- 职业：研究生、咨询顾问、行业分析师
- 使用场景：大量播客内容的主题研究与归档

**核心需求**：
- 需要批量处理播客内容，提取结构化信息
- 需要对内容进行全文检索和关键词定位
- 需要可导出的格式用于后续分析

### 2.2 用户需求优先级

| 需求 | 优先级 | 说明 |
|-----|-------|------|
| 音频上传与语音转写 | P0 | 核心链路第一步 |
| AI 内容理解与提炼 | P0 | 核心价值所在 |
| 多风格笔记模板 | P0 | 满足不同内容场景 |
| 图文笔记生成 | P0 | 区别于竞品的差异化功能 |
| 来源标注与合规 | P0 | 避免版权风险 |
| Web 管理后台 | P1 | 提升使用体验 |
| 播客搜索与发现 | P1 | 扩展内容来源 |
| 历史记录与管理 | P1 | 内容资产管理 |
| 多语言支持 | P2 | 扩展性需求 |
| 批量处理 | P2 | 效率需求 |
| 自动发布 | P3 | 长期目标 |

### 3.2 详细竞品分析

#### 竞品一：AI-Media2Doc（最相似）

**项目地址**：[github.com/hanshuaikang/AI-Media2Doc](https://github.com/hanshuaikang/AI-Media2Doc)

**产品定位**：视频/音频转多种风格文档的 AI 工具

**核心功能**：
- 支持视频和音频文件上传（MP4/MP3等）
- 前端 ffmpeg.wasm 处理，无需本地安装 ffmpeg
- 输出风格：小红书笔记/公众号推文/知识卡片/思维导图/内容总结/字幕文件
- AI 二次问答功能
- 智能截图插入（基于字幕信息自动截图并配图）
- 支持自定义 Prompt
- Docker 一键部署

**技术栈**：Vue 前端 + Python 后端 + Docker

**与本项目的差异**：
| 维度 | AI-Media2Doc | 本项目 |
|-----|-------------|-------|
| 输入源 | 视频/音频文件 | 播客 RSS 订阅/音频文件 |
| 内容理解 | 通用提炼 | 播客场景深度优化（章节划分、金句提取、因果链分析） |
| 模板体系 | 5种输出风格 | 9种笔记模板（v1-v9），覆盖从标准到深度分析的完整光谱 |
| 图文生成 | 智能截图插入 | 基于 HTML 模板渲染的精美图片（3种视觉风格） |
| 设计系统 | 未提供 | 完整的设计 Token 三层架构（Primitive→Semantic→Component） |
| Web 后台 | 基础转换界面 | 功能丰富的管理后台（搜索、库、设置、转录查看器） |

**竞争优势**：
- 更聚焦中文播客场景（支持小宇宙等国内播客平台）
- 更丰富的笔记模板体系
- 更专业的图文设计能力
- 更完善的 Web 管理后台

#### 竞品二：Castmagic

**产品定位**：播客内容的一站式 AI 转化平台

**核心功能**：
- 支持从 YouTube、TikTok、Instagram 等多平台导入内容
- 60+ 语言转录，自动说话人识别
- 从单次录音生成 100+ 种内容资产
- Magic Chat AI 助手（可交互问答）
- 音频剪辑和短视频生成
- 团队协作和审批流程

**定价**：$39/月起

**与本项目的差异**：
- Castmagic 是商业 SaaS 产品，面向英语市场
- 功能更全面但价格昂贵
- 不支持中文内容生态（小红书、公众号等）
- 不提供本地部署选项

**借鉴意义**：
- 多平台内容导入思路值得参考
- AI 对话式内容细化是一个值得探索的方向

#### 竞品三：SummaryYou

**产品定位**：音频/视频转文字 + AI 摘要

**核心功能**：
- 支持 Whisper 本地转录或 Google Cloud API
- 分块处理长文本，支持递归摘要
- 多 LLM 后端支持（GPT/Claude/Ollama）
- 时间戳索引
- 关键要点和决策提取

**与本项目的差异**：
- SummaryYou 侧重摘要而非笔记创作
- 不支持风格化模板输出
- 不支持图文生成

**借鉴意义**：
- 递归分块摘要策略适合处理超长音频
- 多层提炼流程（提取关键点→归纳整合→结构化输出）值得参考

## 四、功能模块设计

### 4.1 功能架构总览

```
播客笔记系统
├── 内容输入层
│   ├── RSS 订阅源解析（小宇宙/Apple Podcasts/通用RSS）
│   ├── 本地音频文件上传
│   └── 音频直链输入
├── 处理引擎层
│   ├── 语音转写（Whisper / Faster-Whisper）
│   ├── 文本预处理（去口头禅/合并短句/语言检测）
│   ├── 内容理解（章节划分/关键信息提取/金句提炼）
│   ├── 笔记生成（多模板 LLM 生成）
│   └── 图文渲染（HTML 模板 → 图片）
├── 输出层
│   ├── 文字笔记（Markdown 格式）
│   ├── 图文笔记（PNG 图片，900×1200px）
│   └── 转录文本（带时间戳）
└── 管理后台层
    ├── 播客搜索与发现
    ├── 历史记录与库管理
    ├── 转录查看器
    └── 系统设置
```

### 4.2 核心功能模块详细设计

#### 模块一：音频输入

**功能描述**：支持多种音频输入方式

| 子功能 | 描述 | 优先级 |
|-------|------|-------|
| RSS 订阅解析 | 输入播客 RSS 链接，自动获取最新单集 | P0 |
| 本地文件上传 | 支持 MP3/WAV/M4A 等常见音频格式 | P0 |
| 音频直链 | 直接输入音频 URL 进行解析 | P1 |
| 多平台适配 | 支持小宇宙、Apple Podcasts 等平台链接 | P1 |

**交互流程**：
```
用户选择输入方式 → 输入 RSS 链接/上传文件/粘贴直链 → 系统解析音频元数据 → 确认开始处理
```

#### 模块二：语音转写

**功能描述**：将音频转化为带时间戳的文字稿

| 子功能 | 描述 | 优先级 |
|-------|------|-------|
| Whisper 转写 | 基于 OpenAI Whisper 模型的语音转文字 | P0 |
| Faster-Whisper | 优化版转写引擎，速度提升 4-5 倍 | P1 |
| 说话人识别 | 区分不同说话人（播客场景特有需求） | P2 |
| 语言检测 | 自动检测音频语言并选择对应模型 | P1 |
| 多语言混排 | 支持中英混合内容的高质量转写 | P1 |

**技术参数**：
- 支持模型：tiny/base/small/medium/large
- 中文优化：支持中文为主的播客内容
- 输出格式：带时间戳的逐句文字稿

#### 模块三：内容理解

**功能描述**：对转写文本进行深度结构化理解

| 子功能 | 描述 | 优先级 |
|-------|------|-------|
| 文本清洗 | 去除口头禅（嗯、啊、那个）、合并短句 | P0 |
| 章节划分 | 按内容逻辑自动划分章节 | P0 |
| 关键信息提取 | 提取核心观点、关键数据、重要案例 | P0 |
| 金句提炼 | 提取有价值的引用语句 | P0 |
| 标签生成 | 自动生成适合多平台的话题标签 | P0 |
| 因果链分析 | 识别内容中的因果逻辑关系（v9 模板特有） | P2 |

**处理流程**：
```
原始转写文本 → 文本清洗 → LLM 理解 → 结构化输出（章节/要点/金句/标签）
```

#### 模块四：笔记生成

**功能描述**：将结构化内容转化为不同风格的高密度社交图文

| 模板 | 风格定位 | 适用场景 | 优先级 |
|-----|---------|---------|-------|
| v1 标准型 | 平衡信息量与可读性 | 通用场景 | P0 |
| v2 深度干货型 | 知识密集、逻辑严密 | 商业/科技/理财类 | P0 |
| v3 故事共鸣型 | 情感驱动、引发共鸣 | 生活/情感类播客 | P0 |
| v4 真人笔记型 | 降低 AI 感，更接近真人笔记 | 追求真实感的用户 | P1 |
| v5 知识翻译官型 | 将专业知识通俗化表达 | 复杂话题的通俗解读 | P1 |
| v6 故事型 | 叙事驱动的笔记结构 | 适合故事性强的内容 | P1 |
| v7 图文结构化型 | 适配图文排版的结构化输出 | 图文笔记生成 | P0 |
| v7d 图文高密度型 | 信息密度更高的图文排版 | 内容量大的场景 | P1 |
| v8 播客凝练版 | 保留播客对话质感的文字稿 | 需要保留对话感的场景 | P1 |
| v9 深度分析型 | 时间线+阶段划分+因果链呈现 | 深度分析类播客 | P0 |

**生成流程**：
```
结构化内容 + 模板 → Jinja2 渲染 Prompt → LLM 生成笔记 → 质量检查 → 保存
```

#### 模块五：图文笔记生成

**功能描述**：将笔记内容渲染为精美的多场景社交图文（高颜值知识卡片）

| 子功能 | 描述 | 优先级 |
|-------|------|-------|
| 封面生成 | 自动生成吸引眼球的封面图 | P0 |
| 内容页生成 | 将笔记内容按模板渲染为多页图片 | P0 |
| 总结页生成 | 生成总结感悟页面 | P0 |
| 思考页生成 | 生成独立思考与见解页面（v9 特有） | P2 |
| 多种视觉风格 | 极简知识感/活力社交感/高级杂志感 | P0 |
| 设计 Token 系统 | 三层架构（Primitive→Semantic→Component） | P0 |

**图片规格**：
- 画布尺寸：900px × 1200px（3:4 比例）
- 安全边距：左右 60px，上下 40px
- 输出格式：PNG

#### 模块六：来源标注与合规

**功能描述**：自动添加来源信息，确保内容合规

| 子功能 | 描述 | 优先级 |

### 6.3 异常处理流程

| 异常类型 | 处理方式 | 用户提示 |
|---------|---------|---------|
| 音频无法下载 | 重试 3 次后失败 | "无法获取音频，请检查链接是否有效" |
| 转写失败 | 记录日志，提示用户 | "语音转写失败，请尝试更换模型或稍后重试" |
| LLM 超时 | 自动重试，增加超时时间 | "AI 处理中，请稍候..." |
| LLM 返回格式错误 | 使用 Fallback 方案 | "已使用备用方案处理" |
| 图文渲染失败 | 跳过图片生成，保留文字笔记 | "图文生成失败，文字笔记已保存" |
| 存储空间不足 | 提示清理 | "存储空间不足，请清理历史文件" |

## 九、风险与应对

### 9.1 技术风险

| 风险 | 影响 | 应对措施 |
|-----|------|---------|
| Whisper 模型资源消耗大 | 低配置设备无法使用大模型 | 提供多模型选择，自动根据硬件推荐 |
| LLM API 调用失败 | 内容生成中断 | 多级重试 + Fallback 方案 |
| HTML 渲染兼容性 | 图文笔记在不同环境下显示不一致 | 使用 Playwright 统一渲染环境 |
| 播客平台 API 变更 | RSS 解析失效 | 维护多平台适配器，快速适配变更 |

### 9.2 产品风险

| 风险 | 影响 | 应对措施 |
|-----|------|---------|
| 笔记模板同质化 | 用户流失 | 持续迭代模板，增加差异化 |
| 中文转写准确率不足 | 用户体验差 | 优先优化中文模型，提供手动修正 |
| 竞品功能快速迭代 | 失去竞争优势 | 保持敏捷迭代，聚焦核心差异化 |

### 9.3 合规风险

| 风险 | 影响 | 应对措施 |
|-----|------|---------|
| 播客内容版权 | 侵权风险 | 自动添加来源标注，限制商用 |
| AI 生成内容标识 | 平台审核风险 | 明确标注 AI 辅助生成 |
| 用户隐私数据 | 数据泄露风险 | 本地部署，数据不出本地 |
