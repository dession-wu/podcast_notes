# UI/UX 修复计划

> **目标**：系统性修复导航功能缺失、转录音频页面排版问题、转录功能异常三大用户反馈问题。

---

## 一、问题诊断总结

### 1.1 导航功能缺失

**现状**：
- 应用使用 Next.js App Router + `next/link` 进行页面跳转
- **无浏览器历史导航控制**（前进/后退按钮）
- 用户只能依赖侧边栏导航，无法使用浏览器原生导航

**影响**：
- 用户体验不完整，不符合现代 Web 应用习惯
- 深层页面返回需要多次点击侧边栏

### 1.2 转录音频页面（Library）问题

**现状**：
- 四个标签页（"全部"/"音频"/"转录"/"图片"）已实现
- 文件列表使用 `AnimatePresence` + `motion.div` 动画
- **关键问题**：`key={file.id}` 可能不唯一（相同播客多集导致重复 key）

**用户反馈问题**：
- 模块切换时下方播客列表排版错误（元素错位、重叠、布局混乱）
- 数据不一致（各模块显示数据不一致）

**技术分析**：
- `library.py` 中 `id=f"{file_type}_{entry.stem}"` 生成逻辑可能导致重复
- 文件列表动画在快速切换标签时可能产生状态冲突
- `files` 状态在标签切换时完全替换，但动画过渡可能未正确处理

### 1.3 转录功能异常

**现状**：
- 转录按钮已集成到音频文件卡片
- 后端 API 已支持智能路由（SenseVoice/Whisper）
- 进度轮询机制已实现（每 2 秒）

**潜在问题**：
- 转录任务状态存储在组件本地状态，页面刷新后丢失
- 错误处理仅显示简单文本，无重试建议
- 转录结果无法持久化查看

## 二、修复方案

### 2.1 导航功能修复

**方案**：在 `TopBar` 组件中添加浏览器历史导航按钮

```
┌─────────────────────────────────────────────────────────┐
│ [←] [→]  Podcast Notes                    [用户头像]   │
└─────────────────────────────────────────────────────────┘
```

**实现要点**：
1. 使用 `window.history.back()` / `window.history.forward()`
2. 监听 `popstate` 事件更新按钮可用状态
3. 样式匹配现有暗色主题设计
4. 在 `TopBar` 左侧添加，与页面标题保持合理间距

**文件变更**：
- `web-dashboard/src/components/TopBar.tsx` — 添加导航按钮

### 2.2 Library 页面修复

#### A. 修复 Key 重复问题

**问题**：相同播客名称 + 相同单集标题会导致 `id` 重复

**修复**：
```typescript
// 当前
id: f"{file_type}_{entry.stem}"

// 修复后
id: f"{file_type}_{entry.stem}_{entry.stat().st_mtime}"
// 或使用完整文件名 + 路径哈希
```

#### B. 修复标签切换动画冲突

**问题**：`AnimatePresence` 在快速切换时可能产生动画队列冲突

**修复**：
1. 添加 `mode="wait"` 到 `AnimatePresence`
2. 为每个标签页添加独立的动画 key
3. 优化文件列表渲染，避免不必要的重渲染

#### C. 修复数据一致性

**问题**：标签切换时 `files` 状态更新与 `typeCounts` 可能不同步

**修复**：
1. 确保 `fetchFiles` 同时更新 `files` 和 `typeCounts`
2. 添加加载状态锁定，避免并发请求导致数据混乱

**文件变更**：
- `web-dashboard/src/app/dashboard/library/page.tsx` — 修复 key、动画、数据同步
- `backend/routers/library.py` — 确保 ID 唯一性

### 2.3 转录功能修复

#### A. 持久化转录任务状态

**问题**：页面刷新后转录任务状态丢失

**修复**：
1. 使用 `localStorage` 存储进行中的转录任务
2. 页面加载时恢复任务并重新启动轮询

#### B. 增强错误处理

**问题**：错误提示过于简单

**修复**：
1. 根据错误类型显示不同的提示信息
2. 提供具体的解决建议（如"文件过大，请尝试分段上传"）
3. 添加"查看详情"展开错误信息

#### C. 转录结果持久化

**问题**：转录完成后结果仅在内存中

**修复**：
1. 将转录结果保存到 `localStorage`
2. 提供"转录历史"查看功能
3. 支持重新使用历史转录结果

**文件变更**：
- `web-dashboard/src/app/dashboard/library/page.tsx` — 增强转录状态管理
- `web-dashboard/src/lib/transcriptionStorage.ts` — 新增持久化存储工具
