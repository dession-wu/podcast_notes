# 播客内容凝练版文字稿 — 图片内容重设计计划

> **目标**：将图片内容重新定位为「播客内容的凝练版文字稿」，严格按播客原始内容结构分标题、分部分记录，突出文字信息展示，确保忠于原意且信息丰富。

---

## 一、当前问题分析

### 1.1 现有内容页的问题

| 问题 | 说明 | 影响 |
|------|------|------|
| **信息太少** | 每页仅 2 个要点，大量留白 | 用户获取信息效率低 |
| **结构扁平** | 所有内容平铺为"核心观点" | 无法体现播客的章节结构 |
| **丢失原始结构** | 未按播客实际讨论顺序组织 | 用户无法对照原内容定位 |
| **金句喧宾夺主** | 金句框占据大量空间 | 压缩了实质内容空间 |
| **类比冗余** | 每个要点强制配类比 | 增加字数但未增加信息量 |

### 1.2 与目标定位的差距

**目标定位**：播客内容的凝练版文字稿
- 应该像「精简后的会议纪要」
- 应该保留播客的章节结构和讨论顺序
- 应该用标题层级清晰划分各部分
- 应该用文字密度填满页面，减少留白

**当前状态**：更像「知识卡片」而非「文字稿」
- 碎片化呈现，缺乏上下文
- 过度设计（卡片、图标、类比）占用内容空间
- 每页信息承载量低

---

## 二、新设计规范

### 2.1 内容定位

```
图片内容 = 播客凝练版文字稿
         = 按播客原始结构分段
         + 保留关键论述和逻辑链
         + 用阴影/加粗突出核心信息
         + 删减口语化填充和过渡语句
```

### 2.2 页面结构（内容页）

```
┌─────────────────────────────────────────┐
│  顶部：章节标题（播客原始讨论主题）          │
│  ─────────────────────────────────────  │
│                                         │
│  【小标题1】播客中的子话题                  │
│  • 关键论述1（保留核心逻辑）                │
│  • 关键论述2（数据/案例保留）               │
│  • 关键论述3（阴影标注核心观点）            │
│                                         │
│  【小标题2】下一个子话题                    │
│  • 关键论述...                           │
│                                         │
│  ─────────────────────────────────────  │
│  底部：页码 + 播客来源标注                 │
└─────────────────────────────────────────┘
```

### 2.3 视觉设计规范

**文字层级**：
| 层级 | 用途 | 字号 | 样式 |
|------|------|------|------|
| H1 | 章节大标题 | 28px | 粗体，主色 |
| H2 | 子话题标题 | 22px | 粗体，深色 |
| Body | 论述内容 | 17-18px | 常规，行高1.6 |
| Highlight | 核心观点/数据 | 17-18px | **阴影背景标注** |
| Caption | 来源标注 | 13px | 浅色 |

**阴影标注效果**（重点强化）：
```css
.highlight-text {
    background: linear-gradient(120deg, #fef3c7 0%, #fef3c7 100%);
    background-repeat: no-repeat;
    background-size: 100% 45%;
    background-position: 0 88%;
    padding: 0 3px;
    font-weight: 600;
}
```

**页面布局**：
- 边距：左右 45px，上下 40px（比当前更窄，增加内容区）
- 行距：1.6（紧凑但可读）
- 段间距：12px（减少空白）
- 每页目标：**4-6 个子话题** 或 **6-8 条关键论述**

### 2.4 内容提取规范（Prompt 要求）

**对 LLM 的新要求**：

```markdown
## 内容提取要求

### 结构忠实度
- 严格按照播客的讨论顺序组织内容
- 保留播客中的章节/话题划分
- 每个章节提取 3-5 个关键论述点

### 信息密度
- 每页内容应填满 80% 以上页面空间
- 单条论述控制在 30-50 字
- 优先保留：核心观点、关键数据、重要案例、逻辑结论
- 必须删除：口语化填充、过渡语句、重复表达、闲聊内容

### 突出标注规则
- 核心观点：黄色阴影背景标注
- 关键数据：加粗 + 阴影标注
- 反常识结论：加粗 + 阴影标注
- 方法论/步骤：编号列表 + 阴影标注

### 输出格式
{
  "sections": [
    {
      "section_title": "播客中的章节标题",
      "subsections": [
        {
          "subtitle": "子话题",
          "points": [
            {
              "text": "论述内容（30-50字）",
              "highlight": "需要阴影标注的核心片段",
              "is_key_data": false
            }
          ]
        }
      ]
    }
  ]
}
```

## 三、模板重新设计

### 3.1 新内容页模板（content_transcript.html）

**设计要点**：
- 去除卡片式布局，改用**连续文本流**
- 去除 💡 图标和类比框，释放空间
- 增加**多级标题**体现结构
- 用**列表符号**组织论述点
- 阴影标注直接嵌入文本流

```html
<div class="content-page">
  <!-- 章节标题 -->
  <h1 class="chapter-title">第一章：什么是有效前沿</h1>
  
  <!-- 子话题 -->
  <div class="subsection">
    <h2 class="subtitle">有效前沿的定义</h2>
    <ul class="point-list">
      <li>有效前沿是风险和收益坐标系中的<span class="highlight">"最佳曲线"</span></li>
      <li>在同样风险下，曲线上的组合收益最高</li>
      <li>在同样收益下，曲线上的组合风险最低</li>
    </ul>
  </div>
  
  <!-- 子话题 -->
  <div class="subsection">
    <h2 class="subtitle">实际意义</h2>
    <ul class="point-list">
      <li>曲线以下的组合都是<span class="highlight">"性价比低"</span>的垃圾组合</li>
      <li>投资者应该追求曲线上的配置方案</li>
    </ul>
  </div>
  
  <!-- 底部 -->
  <div class="footer">...</div>
</div>
```

### 3.2 封面页调整

- 保持现有设计（标题 + 播客信息）
- 增加副标题提示：「播客内容凝练笔记」

### 3.3 总结页调整

- 从「清单式」改为「结构化复盘」
- 保留章节结构，每个章节列出 2-3 个核心 takeaway
- 增加「建议收听原播客」的引导

## 四、Prompt 重新设计

### 4.1 新 Prompt 核心变化

| 维度 | 旧 Prompt (v7d) | 新 Prompt (v8_transcript) |
|------|------------------|---------------------------|
| **定位** | 小红书干货笔记 | 播客凝练版文字稿 |
| **结构** | 扁平要点列表 | 按播客章节分层 |
| **输出** | title/content/highlight/analogy | section/subtitle/points |
| **类比** | 强制每个要点配类比 | **删除类比**，释放空间 |
| **金句** | 独立金句框 | 融入文本流，加粗标注 |
| **密度** | 每页2个要点 | 每页4-6个子话题 |

### 4.2 Prompt 关键指令

```markdown
# 核心任务

将播客转录文本整理为「凝练版文字稿」，按播客原始讨论结构分章节、分子话题记录。

# 输出要求

1. **结构忠实**：严格按照播客的讨论顺序和章节划分
2. **信息完整**：每个子话题提取 3-5 个关键论述，不遗漏核心逻辑
3. **删减精简**：删除口语化填充、过渡语句、重复内容
4. **突出标注**：核心观点、关键数据用阴影效果标注
5. **无类比**：不需要生活化类比，直接呈现原内容
6. **无金句框**：金句融入文本流，用加粗或阴影标注

# 输出格式

{
  "sections": [
    {
      "section_title": "章节标题（来自播客讨论主题）",
      "subsections": [
        {
          "subtitle": "子话题",
          "points": [
            {
              "text": "论述内容",
              "highlight": "需要阴影标注的核心片段（可选）"
            }
          ]
        }
      ]
    }
  ]
}
```

## 五、实施步骤

### Phase 1：Prompt 重新设计（1-2小时）

1. **创建 v8_transcript Prompt**
   - 按章节结构输出
   - 删除类比要求
   - 删除独立金句框
   - 增加结构忠实度要求

2. **修改 ContentProcessor**
   - 添加 v8 模板映射
   - 修改 `_extract_structured_content` 解析新格式

### Phase 2：HTML 模板重新设计（2-3小时）

1. **创建 content_transcript.html**
   - 连续文本流布局
   - 多级标题样式
   - 列表符号样式
   - 阴影标注样式
   - 紧凑边距和行距

2. **调整 ImageGenerator**
   - 支持新模板渲染
   - 每页容纳更多内容（4-6个子话题）

3. **调整封面和总结页**
   - 封面增加「凝练笔记」标识
   - 总结页改为结构化复盘

### Phase 3：测试与验证（1-2小时）

1. **生成测试笔记**
   - 使用真实播客内容
   - 对比新旧版本的信息密度

2. **验证结构忠实度**
   - 检查是否按播客章节组织
   - 检查关键信息是否遗漏

## 六、预期效果

### 6.1 信息密度对比

| 指标 | 当前 (v7d) | 目标 (v8) |
|------|-----------|-----------|
| 每页子话题数 | 2 | 4-6 |
| 每页论述点数 | 2-3 | 8-12 |
| 页面留白率 | ~40% | ~15% |
| 章节结构 | 无 | 完整保留 |
| 内容忠实度 | 提炼后重组 | 按原结构凝练 |

### 6.2 用户体验提升

- **对照原播客**：用户可以根据图片上的章节标题，快速定位到播客的对应部分
- **信息获取效率**：一页获取的信息量翻倍
- **收藏价值**：更像一份精简的会议纪要，适合收藏和回顾

---

## 七、风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 文字过密影响可读性 | 中 | 中 | 调整字号和行距，确保最小可读性 |
| LLM 不按章节结构输出 | 中 | 高 | Prompt 中增加章节划分示例 |
| 播客本身结构不清晰 | 低 | 中 | 自动识别话题转换点作为分段依据 |
| 阴影标注过多 | 低 | 低 | 限制每页阴影标注数量 |
